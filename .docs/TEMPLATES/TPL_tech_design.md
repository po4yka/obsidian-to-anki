# Technical Design Template

## Metadata
- **ID:** `TD-XXX`
- **Title:** _Component / feature_
- **Author:** _Name_
- **Date:** _YYYY-MM-DD_
- **Status:** Draft / In Review / Approved

## Context
- Problem summary
- Links to requirements / proposals

## Objectives & Non-Goals
- Bullet list of what this design covers
- Explicit exclusions

## Architecture
- High-level diagram description
- Components and responsibilities
- Data models / schemas

## Detailed Design
### Flow / Sequence
- Step-by-step narrative

### Algorithms / Logic
- Pseudocode or references

### API / Interface Changes
- New or updated endpoints/classes with signatures

## Deployment & Migration
- Feature flags
- Data backfill or schema changes

## Observability
- Metrics, logs, traces

## Security & Privacy
- Threat considerations
- Access control

## Risks & Mitigations
- Table of risk, impact, mitigation

## Testing Strategy
- Unit, integration, e2e
- Test data / fixtures

## Rollout Plan
- Incremental steps
- Owner(s)

## Open Questions
- Items needing follow-up
